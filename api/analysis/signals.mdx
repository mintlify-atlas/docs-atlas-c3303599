## Overview

The trading signals endpoint analyzes market data using multiple technical indicators to generate actionable BUY, SELL, or HOLD signals. Signals are generated by combining RSI, Moving Averages, MACD, and Bollinger Bands analysis.

## Query Parameters

<ParamField query="symbol" type="string">
  The trading pair symbol to analyze (e.g., "BTC/USDT"). If omitted, returns signals for all active trading pairs.
  
  Example: `BTC/USDT`
</ParamField>

<ParamField query="exchange" type="string">
  The exchange name where the symbol is traded (e.g., "binance"). Required if `symbol` is specified.
  
  Example: `binance`
</ParamField>

<ParamField query="timeframe" type="string" default="1h">
  The candlestick timeframe for analysis. Common values include:
  - `1m` - 1 minute
  - `5m` - 5 minutes
  - `15m` - 15 minutes
  - `1h` - 1 hour (default)
  - `4h` - 4 hours
  - `1d` - 1 day
</ParamField>

<ParamField query="min_confidence" type="number" default="0.6">
  Minimum confidence threshold for signals (0.0 to 1.0). Only signals with confidence at or above this threshold will be returned.
  
  Example: `0.7` for higher quality signals
</ParamField>

## Response

<ResponseField name="signals" type="TradingSignal[]">
  Array of trading signals generated from technical analysis.
  
  <Expandable title="TradingSignal">
    <ResponseField name="symbol" type="string" required>
      The trading pair identifier (e.g., "BTC/USDT").
    </ResponseField>
    
    <ResponseField name="exchange" type="string" required>
      The exchange name where the signal applies.
    </ResponseField>
    
    <ResponseField name="signal_type" type="string" required>
      The recommended action. One of:
      - `BUY` - Bullish signal, consider entering long position
      - `SELL` - Bearish signal, consider entering short position or closing long
      - `HOLD` - Neutral signal, no clear directional bias
    </ResponseField>
    
    <ResponseField name="strength" type="string" required>
      Signal strength classification:
      - `WEAK` - Confidence < 0.6
      - `MODERATE` - Confidence 0.6 - 0.8
      - `STRONG` - Confidence >= 0.8
    </ResponseField>
    
    <ResponseField name="price" type="number" required>
      The current price at which the signal was generated.
    </ResponseField>
    
    <ResponseField name="reason" type="string" required>
      Human-readable explanation of why the signal was generated.
      
      Example: `"RSI indicates oversold conditions; Price above moving averages (bullish trend)"`
    </ResponseField>
    
    <ResponseField name="confidence" type="number" required>
      Numerical confidence score from 0.0 to 1.0 indicating the algorithm's certainty in the signal.
    </ResponseField>
    
    <ResponseField name="timestamp" type="string" required>
      ISO 8601 timestamp when the signal was generated.
    </ResponseField>
    
    <ResponseField name="indicators_used" type="string[]" required>
      List of technical indicators that contributed to this signal.
      
      Common indicators include:
      - `RSI` - Relative Strength Index
      - `SMA` - Simple Moving Average
      - `EMA` - Exponential Moving Average
      - `MACD` - Moving Average Convergence Divergence
      - `Bollinger Bands` - Volatility-based bands
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="count" type="integer" required>
  The number of signals returned in the response.
</ResponseField>

<ResponseField name="timestamp" type="string" required>
  ISO 8601 timestamp when the response was generated.
</ResponseField>

## Signal Generation Logic

<Info>
  Signals are generated by analyzing multiple technical indicators:
  
  **RSI Analysis:**
  - RSI < 30: Oversold (BUY signal)
  - RSI > 70: Overbought (SELL signal)
  
  **Moving Average Analysis:**
  - Price > SMA20 > SMA50: Bullish trend (BUY)
  - Price < SMA20 < SMA50: Bearish trend (SELL)
  
  **Bollinger Bands:**
  - Price near lower band: Potential bounce (BUY)
  - Price near upper band: Potential reversal (SELL)
  
  The final signal aggregates these indicators with weighted confidence scoring.
</Info>

## Examples

<CodeGroup>

```bash Single Symbol

  -H "Authorization: Bearer YOUR_API_KEY"
```

```bash All Active Pairs
curl -X GET "https://api.neuratrade.com/api/v1/analysis/signals?min_confidence=0.7" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## Best Practices

<Warning>
  **Trading Signals Are Not Financial Advice**
  
  These signals are generated by automated technical analysis and should not be considered financial advice. Always:
  - Perform your own research
  - Consider multiple data sources
  - Implement proper risk management
  - Never trade more than you can afford to lose
</Warning>

<Tip>
  **Optimal Usage**
  
  - Use `min_confidence=0.7` or higher for higher quality signals
  - Combine multiple timeframes for better confirmation
  - Monitor signal changes over time rather than acting on single signals
  - Consider signal strength in addition to signal type
</Tip>
